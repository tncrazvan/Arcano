About ElkServer

ElkServer is a simple Java servlet manager.
It provides a few tools to help you create your own servlet.

Serving files

Like any web server, ElkServer can serve media and text files across Http.
Given a file request, the server will look for the file inside the public directory and serve the file back to the client if it exists.

The public directory is the directory that your server is serving.
The name of this directory is fetched by the server from ElkServer#listen(String[] args), specifically, it is the first elements or args.
In order to start serving files, all you need to do is run java -jar ElkServer.jar </path/to/public/dir> <port_number> from your terminal.

Controllers

In addition to files, you can use controllers to add more depth to your server.
While serving files to your clients is nice, controllers are an easy way to achieve a higher level of abstraction.

Think of a controller as an object which is able (not required) to provide an output given an input.
The input is fetched from the client's HTTP request, the controllers executes its job, and the output is then fed back to the client.

Controller example

In this example I'll create a controller which will receive 2 variables in input, it will then sum the two variables if they are numeric, and then return the result back to the client.


  public class Test implements HttpInterface{

    @Override
    public void main(HttpEvent e, ArrayList get_data, HashMap post_data) {
      //fetch the url query variable "a"
      int a = Integer.parseInt(e.getUrlQuery("a"));
      //fetch the url query variable "b"
      int b = Integer.parseInt(e.getUrlQuery("b"));
      //sum and send back to client
      e.send(a+b);
    }

    @Override
    public void onClose() {
      //do something when connection ends (maybe close some Database connection?)
    }

  }
